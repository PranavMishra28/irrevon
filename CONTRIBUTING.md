# Contributing to Irrevon

**Short version: not yet.** This repository is licensed under Apache-2.0 (see
[LICENSING.md](LICENSING.md) and
[ADR-0028](docs/decisions/0028-apache-2-license.md)), but outside code and content
contributions are not yet accepted.

- **Pull requests with code or content cannot be merged.** The outbound Apache-2.0
  license lets people use, modify, and redistribute the published work; it does not
  establish the project's inbound contribution process. That governance remains open
  in [ADR-0014](docs/decisions/0014-licensing.md). PRs will be closed with a pointer
  to this file.
- **Issues are welcome** for defect reports and questions. By filing an issue you
  agree its text may inform the project. Do not paste code you consider yours.
- **Suspected vulnerabilities must not be filed publicly.** Follow the conditional
  private-reporting guidance in [SECURITY.md](SECURITY.md).

## Why contributions remain closed

ADR-0028 resolved only the repository's **outbound license**. ADR-0014's
contributor-governance half is still open: the owner has not ratified an inbound
policy, enforcement mechanism, or engine-specific acceptance policy. This file does
not choose those mechanisms on the owner's behalf.

When that human decision is ratified, this guide and the repository's enforcement
settings must be updated before any outside pull request is accepted.

Until then, the most useful participation is running the demo, filing precise defect
reports or questions without contributed code, and citing the benchmark
preregistration in review.
