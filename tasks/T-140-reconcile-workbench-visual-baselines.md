## Objective

Inspect every current `make web-vrt` difference and update the exact Linux
baselines only if all differences are intentional, visually sound consequences
of already-reviewed text or public-truth changes.

## Scope

**Allowed to write:**

- this task file;
- `web/e2e/visual/__baselines__/**`;
- ignored Workbench VRT output artifacts generated by the existing test tooling.

**Forbidden:** every other tracked file; application code; fixtures; thresholds;
screenshot masks; test configuration; implementation changes; commits; external
mutations; deployment; and publication.

## Acceptance criteria

- [x] Every one of the 18 reported visual differences is inspected and explained.
- [x] No accepted difference is caused by layout regression, missing assets, font
      drift, rendering nondeterminism, or weakened visual-test configuration.
- [x] Exact Linux baselines are refreshed through the existing Playwright snapshot
      mechanism only if all differences are intentional and visually sound.
- [x] Representative refreshed actuals are visually inspected.
- [x] `make web-vrt` passes twice consecutively after any baseline refresh.
- [x] Workbench static and browser checks pass.
- [x] `git diff --check` passes.
- [x] No code, fixture, threshold, mask, or test-configuration file is changed.

## Required validation

```text
make web-vrt
make web-vrt
make web-check
make web-test
git diff --check
```

## Human review triggers — stop and report if:

- any difference cannot be traced to an intentional current text/truth change;
- any screenshot shows layout regression, missing assets, font drift, clipping,
  overlap, responsive breakage, or nondeterminism;
- exact baseline regeneration requires an out-of-scope change.

## Definition of done

All 18 differences are explained and visually sound, exact Linux baselines are
reconciled through the existing snapshot mechanism, required validation passes
twice where specified, no forbidden file changes occur, and this task is marked
`done`.

## Outcome

All 18 differences were intentional consequences of already-reviewed source
changes made after the previous baseline revision:

- Eight Overview snapshots (light/dark at 1280, 1600, 375, and 768) replace the
  obsolete statement that no benchmark contract exists with the truthful
  disclosure that contracts and a harness exist, developmental S-REF pilots are
  non-confirmatory mechanism evidence, and no scientific or confirmatory result
  is shown.
- Eight Demo snapshots (light/dark at 1280, 1600, 375, and 768) replace
  “strongest conventional baseline” with the accurate “developmental B5
  file-journal stand-in” description.
- Two mobile Attention snapshots (light/dark at 375) reflect the reviewed
  `break-all` containment safeguard for the long machine identifier.

Pixel-bound inspection localized every difference to those three regions.
Desktop, tablet, phone, light, and dark actuals showed no missing assets, font
drift, overlap, clipping, unrelated layout movement, color drift, or responsive
regression. Playwright's existing pinned Linux snapshot mechanism regenerated
exactly those 18 files; no code, fixture, threshold, mask, or configuration was
changed.
