# Support

Irrevon is a solo-maintained open-source research preview. Support is
best-effort; no response time, uptime, consulting availability, or commercial
service is promised.

Use the channel that matches the request:

| Need | Channel |
|---|---|
| Reproducible defect | [GitHub bug-report form](https://github.com/PranavMishra28/irrevon/issues/new?template=bug.yml) |
| Documentation problem | [GitHub documentation form](https://github.com/PranavMishra28/irrevon/issues/new?template=documentation.yml) |
| Scoped feature or design proposal | [GitHub proposal form](https://github.com/PranavMishra28/irrevon/issues/new?template=proposal.yml) |
| Usage question | [GitHub Discussions — Q&A](https://github.com/PranavMishra28/irrevon/discussions/categories/q-a) |
| Open-ended community conversation | [GitHub Discussions](https://github.com/PranavMishra28/irrevon/discussions) |
| Early idea or product feedback | [GitHub Discussions — Ideas](https://github.com/PranavMishra28/irrevon/discussions/categories/ideas) |
| Benchmark-integrity concern | [GitHub benchmark-integrity form](https://github.com/PranavMishra28/irrevon/issues/new?template=benchmark-integrity.yml) |
| Vulnerability | [GitHub private vulnerability reporting](https://github.com/PranavMishra28/irrevon/security/advisories/new); see [SECURITY.md](SECURITY.md) |
| Code-of-Conduct report | GitHub platform reporting/blocking tools; see [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) |
| Collaboration inquiry | [GitHub proposal form](https://github.com/PranavMishra28/irrevon/issues/new?template=proposal.yml), labeled as an inquiry; no paid offering is implied |

Before filing, search existing issues, run `uv run irrevon doctor`, and remove
credentials, personal data, destination payloads, and production identifiers from
logs. Include the Irrevon version, Python version, operating system, minimal
reproduction, expected behavior, and actual behavior.

Only the latest source revision and supported tagged releases receive fixes.
Compatibility and deprecation policy is documented in
[docs/operations.md](docs/operations.md#compatibility-versioning-and-deprecation).
